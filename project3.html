COMPASS

A MOBILE APPLICATION

TEAM

Terence Liu

SKILLS

JAVA
Android Studio
Sketching

PROBLEM

Time is often wasted while traveling through TransLink, a transit service company, from fumbling for our Compass Cards to getting an accurate departure time. Currently, there is no single application that contains all the necessities to travel around Metro Vancouver through TransLink services. To provide a convenient solution for this problem, I designed a prototype application that contains features like being able to tap your mobile device rather than a Compass Card, showing the location of nearby TransLink services, and providing bus departure times.

DESCRIPTION

The Compass App is a convenient alternative to the Compass Card, allowing users to not only tap their mobile devices to ride on TransLink services, but it also provides a precise calculation of an individual’s arrival time to each station... This feature is geared for a variety of people with different walking speeds. An additional feature includes taking a user’s location and letting the user know where the closest transit station is and what time the bus will arrive at. The Compass App has all the information a user would need to get around Metro Vancouver in one well-packaged app.

PROCESS

Figure 2.2: Prototyping using Flinto to visualize what the website would look like on a laptop.

I began this project by ideating possible solutions for making my everyday lifestyle more convenient. I focused specifically on working with TransLink services, and thought of handy features that can be placed all in one application... Next, I drew up an outline of how I wanted to display the features. Afterwards, I moved onto developing and coding the features which includes displaying nearby transit services, displaying the bus schedule, displaying directions, a prototype for tapping your mobile device instead of the compass card, and it displays the cardinal direction. For displaying nearby transit services and the directions I used the Google Map API, and for displaying the bus departure times I used TransLink’s API. The prototype tap feature is done with NFC by sending a signal from one device to the other to communicate to the user that a tap has occurred. Finally, I brought these features together and created my own app. Additionally, I added visual treatments by picking the typeface, color palette, and layout.

ROLE

In this project, I designed and developed a working prototype of the mobile app that includes all the core features for getting around the TransLink system... I initially ideated by thinking of issues that I wanted to solve in my everyday lifestyle, and this issue is a significant part of my lifestyle. I drew sketches of how I wanted to layout the app and thought of the core features I wanted to include. I programmed on Android Studio each of the core features and brought them all together in one seamless application.

REFLECTION

Working on creating an app for the first time made me reflect on how I could make my app unique from others, as well as how I could create a stronger user experience... I focused on creating a more personalized experience so that each user could have their app more geared towards themselves. Unlike other navigation apps that uses historic data on the average walking speed to calculate estimated arrival times, my app would use the GPS location to constantly calculate velocity, and position for a more accurate estimate on the time of arrival for a person to person basis. This real time data that will update the individuals on how much faster or slower they may need to move to make it on the platform for the next Skytrain or bus. Looking forward, I hope to continue providing more customized experiences for the app users.
